The foundation section contains the building blocks of HubAI. These libraries can reference each other but none of the libraries in the feature section

Each foundation section contains a code and a test section. Testing is and important part of an API First approach and each library should be fully tested.

ExecutionProviders

An execution provider connects to a specific api (or set of apis) to return a result.

IHubAIExecutionProvider: Interface implemented by a HubAI execution provider. This interface will contains:

  • IsMeteredProvider: Returns true if this provider has a potential cost or usage limit related to it
  • ProviderSettings: A collection of settings required by the provider to process the content
  • ErrorMode: This determines how the pipeline handles errors. Possible values are:
    • SKIP_ON_ERROR: If an error occurs in a provider it will log the errror and go onto the next step
    • STOP_ON_ERROR: If an error occurs all processing will stop and an error will be displayed.
    • ProbabilityWeight: This number determines how likely we feel this is best result
  • Validate: Validate The Provider based on content passed in. Skips call to APIs and returns a sample result that may not match content. Any analysis will be written to the AdditionalMeta on the results
  • Process: Processes the content and provides a list of possible results

Pipelines

A pipeline contains a series of steps that may or may not be executed based on criteria/rules to return a series of results.

IHubAIPipeline/HubAIPipeline: A pipeline contains a series of steps that may or may not be executed. This pipeline will contains:

  • PipelineExecutionMode: This determines how pipeline steps are treated. Possible values are:
    • FIRST_MATCHING: The first step that is valid to
    • ALL_MATCHING
    • FIRST_NON_METERED_MATCHING
    • ALL_NON_METERED_MATCHING
  • ErrorMode:This determines how the pipeline step handles errors. Possible values are:
    • SKIP_ON_ERROR: If an error occurs in a provider it will log the errror and go onto the next step
    • STOP_ON_ERROR: If an error occurs all processing will stop and an error will be displayed.
    • USE_PIPELINE_STEP_ERROR_MODE: If an error occurs it will look at the ErrorMode on the PipelineStep to determine the best response.
  • PipelineSteps: This is a step that may be executed based on a set of criteria or rules

IHubAIPipelineStep/HubAIPipelineStep: A pipeline step contains a provider and critera on when to execute it. This pipeline step will contains:

  • ProviderName: The name of the provider to run for this step
  • ProviderWeight: Applied to the result to reflect the importance given to this provider in determining the result
  • ProviderSettings: Additional settings needed to use this provider. For example an API Key.
  • Rules: A list of rules in JSON format that determine when to run it and other factors that would affect the execution
  • ExecutionProvider: The actual provider to run
  • ErrorMode:This determines how the pipeline step handles errors. Possible values are:
    • SKIP_ON_ERROR: If an error occurs in a provider it will log the errror and go onto the next step
    • STOP_ON_ERROR: If an error occurs all processing will stop and an error will be displayed.
    • USE_ERROR_HANDLING_RULE: If an error occurs it will look for the ERROR_HANDLER rule and will use that to determine the response to an error.
    • USE_PROVIDER_STEP_ERROR_MODE: If an error occurs it will look at the ErrorMode on the Provider to determine the best response.

Results

IHubAIResult/HubAIResult: A general HubAI Result is returned from every execution provider. This result will contains:

  • ResultType: The type of result returned eg. LANGUAGE_DETECTION, IMAGE_TAGGING, SENTIMENT, etc
  • Version: This is used in case different providers return different JSON formats for the same Result Type
  • Result: The result in JSON
  • ProbabilityWeight: This number determines how likely we feel this is best result
  • AdditionalMeta: This is inforamation that describes how the result was obtained and if there are any other information known

Serialization

These libraries provide functionality to load and save objects to JSON.

HubAISerialization: This class is responsible for serializing and deserializing objects to JSON. This class contains:

  • DeserializeJsonObject: Deserialize a JSON Content to an object.
  • Version: Serialize object to a JSON stream
